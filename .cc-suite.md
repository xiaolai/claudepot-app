## Audit Focus

quality-first

Additional instructions appended to every audit's developer-instructions:

```text
Prioritize code quality findings. Flag untested critical paths, high cyclomatic complexity, and DRY violations as High regardless of other severity heuristics.

Project-specific additions for the quality bar:
- The four-noun domain model (account, cli, desktop, project) is load-bearing. Flag any code that introduces a new top-level noun as architectural drift.
- `claudepot-core` is pure Rust with NO Tauri dependency. Business logic in `claudepot-cli` or `src-tauri` is a layering violation — flag as High.
- Credentials/tokens must never cross the IPC boundary in serialized form. The pattern is `*_copy` (Rust-side clipboard write + receipt DTO) or `*_add` (paste-in with zeroize). Returning a token string over IPC is a Critical finding.
- Path handling code must use `simplify_windows_path` before `sanitize_path`, never call `canonicalize` directly. Cross-platform regression is a High finding even when tests pass on the local OS.
- React: render-if-nonzero is the rule. `0 sessions · 0 MB` style spans, empty cards, and zero-state chrome that ships are quality-first findings.
- Tokens come from `src/styles/tokens.css`. Hardcoded color / size / radius values in `.css` or inline styles are flagged unless they come from `src/icons.ts` or are pre-existing in untouched files.
```

## Project-Specific Instructions

Custom instructions appended to Codex's developer-instructions for every command.

```text
This is a Rust + Tauri + React control-center app for Claude Code and Claude Desktop. Follow these constraints:

1. `claudepot-core` is the only place business logic lives. `claudepot-cli` and `src-tauri/src/` are thin wrappers.
2. NEVER use the `keyring` crate or `SecItem*` for CC's `Claude Code-credentials` Keychain item — that ONLY goes through `/usr/bin/security` subprocess (see `crates/claudepot-core/src/cli_backend/keychain.rs`). `keyring` is reserved for Claudepot's OWN keychain surface.
3. Two SQLite stores cross await points as `Mutex<Connection>` (accounts.db, sessions.db) — never split the lock across an `.await`.
4. The `dev-docs/` directory is gitignored — never assume a file there will ship with a commit.
5. `web/` is a self-contained Next.js subproject with its own pnpm-lock; run `pnpm` commands from inside `web/`, never from the root.
6. The four nouns in the domain model are account, cli, desktop, project. Surfaces (Sessions, Config, Activities) are presentation over those nouns, not new nouns.
7. Auto-rotation lives in `claudepot_core::rotation`; the orchestrator bridge is `src-tauri/src/rotation_orchestrator.rs`. Hook into `usage_snapshot::run_tick` rather than creating new tick loops.
8. CI runs clippy on Linux with `-D warnings`. The local validators `spark01` (Linux clippy) and `jokershp` (Windows MSVC compile) are the pre-push gates for release tags — see `dev-docs/release-validation.md` if it exists.
```
