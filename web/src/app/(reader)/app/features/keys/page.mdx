# Keys

![ClauDepot — keys tab](/screenshots/keys.png)

Two stores under one tab:

1. **API keys** for the services you use alongside Claude &mdash;
   Anthropic console, Resend, OpenAI, GitHub PATs, OAuth refresh
   tokens. Backed by your OS keychain.
2. **The secret vault** &mdash; a local-only named-secret store for
   the kind of values you keep dropping into a project's `.env` file
   (database URLs, third-party webhook secrets, Stripe test keys).
   Backed by a `0600`-permissioned SQLite store at
   `~/.claudepot/env-vault.db`.

Both share the same security posture: secrets never enter the
renderer's JS heap on the way out, the clipboard self-clears after
30s, and `sk-ant-…` patterns are scrubbed from logs and error
messages.

## Add a key

Three flavours:

- **API key** &mdash; long-lived secret, e.g.
  `sk-ant-api03-…`. ClauDepot stores it under a label you choose.
- **OAuth token pair** &mdash; access + refresh tokens with an
  expiry. ClauDepot tracks expiry and warns before tokens go stale.
- **Personal access token** &mdash; GitHub-style PAT with scopes.

Pasting in a secret happens entirely in Rust on the backend; the value
is zeroized from local memory on every exit path before the IPC
returns. The renderer&rsquo;s React state is cleared in a `finally`
block on the same call.

## Copy

Every key has a *Copy* button. Behaviour:

- ClauDepot writes the secret to your OS clipboard via Tauri&rsquo;s
  clipboard plugin — Rust-side, never returned to the renderer.
- The IPC returns only a receipt: a label, a `sk-ant-…xyz`-style
  preview, and a clear deadline.
- After 30 seconds, ClauDepot reads the clipboard. If it still holds
  the same secret (no other app has overwritten it), ClauDepot
  clears the clipboard. If something else is on the clipboard now,
  ClauDepot leaves it alone.

The result: secrets stay in the clipboard exactly long enough to paste
once, and never leak to a future paste-target.

## Audit

Each row shows last-used (when ClauDepot last copied it for you),
expiry (for OAuth pairs), and the key&rsquo;s scope or label. A keys
event-log records every mint and revoke against the local DB.

## Rotate

Editing a key creates a new keychain entry and atomically swaps it
in. The old entry is deleted only after the new one is verified to
work.

## Secret vault

A second store on the same tab, for the values that don&rsquo;t
belong in the OS keychain because every project wants a different
copy: database URLs, webhook secrets, third-party API tokens that
land in `.env` files. The vault is a local-only SQLite store at
`~/.claudepot/env-vault.db` (mode `0600`); it does not touch the OS
keychain.

The vault is a *movement layer*, not a text editor. Two operations:

- **Copy** &mdash; identical to the API-key copy path. Rust-side
  clipboard write via the Tauri plugin, returns only a receipt, and
  the clipboard self-clears after 30 s if it still holds the same
  value.
- **Inject** &mdash; from a project&rsquo;s **Environment files**
  panel in [Projects](/app/features/projects), pick a vault entry
  and drop it into a chosen `.env*` key without rendering the secret
  anywhere in the UI. The same panel handles comment / uncomment /
  delete on existing keys, all format-preserving (only the target
  line is touched, surrounding whitespace and comments stay).

`.env` filenames are validated server-side (no separators / `..` /
NUL); the renderer cannot supply a path that escapes the project
root.

## Why this matters

Token values starting with `sk-ant-` (or any other secret pattern)
are never logged, never printed in error messages, never serialised
into IPC responses. Truncated previews use the form `sk-ant-oat01-Abc…xyz`
— enough to identify the key visually, not enough to re-use it.
See
[`KeyError`](https://github.com/xiaolai/claudepot-app/blob/main/crates/claudepot-core/src/keys/errors.rs)
for the audited Display impl.
