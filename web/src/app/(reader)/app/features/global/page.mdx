# Global

![ClauDepot — global tab](/screenshots/global.png)

The Global tab is the read-only window onto everything Claude Code
maintains *across all your projects* — config layers, memory files,
the tips ledger CC keeps internally, and the update channel for
both Claude Code and Claude Desktop. Four sub-tabs.

## Config

A tree of every layer Claude Code consults at startup, in cascade
order: managed policy, user prefs, global config, plugins, and any
memory CC pulls in across projects. Each leaf opens to show the raw
file (or the merged effective view, depending on the leaf type), so
when something behaves oddly you can see *which* layer is responsible
without grepping `~/.claude/`.

This is the user-level half of the same Config view that lives
inside Projects → Config; that one is forced to a project anchor.
This one is forced to the global anchor — they share one component,
two scopes.

## Memory

Health cards for your machine-wide CLAUDE.md (`~/.claude/CLAUDE.md`)
and for Claude Code's per-project MEMORY.md indexes. Three signals
per file:

- **Existence** — does the file exist at all?
- **Bloat** — line count plus an estimated token cost.
- **Visibility** — physical lines past CC's truncation cutoff,
  which the model never sees no matter what they say.

Pure read; ClauDepot doesn't presume to suggest what you should cut.
The tab is a diagnostic, not an editor.

## Tips

Claude Code ships with an internal *tips ledger* — short
contextual hints CC schedules and dismisses based on its own state.
The Tips tab lifts that ledger into a searchable, filterable list:
which tips exist, which ones CC has shown you, when, and why. Useful
when you want to see what CC was about to recommend, or to confirm a
tip you saw and want to look up later.

Catalog is extracted from the user's CC binary, joined against
`~/.claude.json`'s `tipsHistory` and ClauDepot's local snapshot log.

## Updates

A unified update manager for both surfaces:

- **Claude Code (CLI)** — installed version, latest available
  version, channel. Updates run via the same npm-global-install path
  CC uses natively.
- **Claude Desktop** — installed version, latest available, channel.
  Updates use the platform-native flow (DMG drop on macOS, MSI on
  Windows).

ClauDepot's *own* updates live in Settings → About, not here. This
panel is about keeping Anthropic's two apps current.

## CLI mirror

Most Global tabs have CLI equivalents:

```bash
claudepot config show            # cascade view
claudepot status                 # version + sanity check across surfaces
claudepot doctor                 # deeper diagnostics
```

The Memory and Tips tabs are GUI-only today.
