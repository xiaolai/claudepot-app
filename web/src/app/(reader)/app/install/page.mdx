# Install

> **Status: beta (`0.2.6`).** Daily-driven on macOS. Linux and
> Windows builds are green but less seasoned.

Two paths: grab a signed installer from the latest release (fast,
recommended), or build from source (one extra step, works on every
platform Rust + Node target).

## Recommended — download a signed installer

Every tagged release publishes installers for macOS, Linux, and
Windows. The download page reads the latest release straight from
GitHub and matches the build to your platform.

[**Open the download page →**](/app/download)

What ships per platform:

| Platform | Artifact |
|---|---|
| macOS (Apple Silicon) | `Claudepot-aarch64.dmg` — drag into Applications |
| macOS (Intel) | `Claudepot-x86_64.dmg` |
| Windows (x64) | `Claudepot-x86_64-setup.exe` (NSIS) or `Claudepot-x86_64.msi` |
| Linux (x64) | `Claudepot-x86_64.AppImage`, `.deb`, or `Claudepot-x86_64-linux.tar.gz` |
| Linux (ARM64) | `claudepot-aarch64-linux.tar.gz` |

Every macOS `.dmg` is signed with a Developer ID certificate and
notarized; every Windows `.msi` carries an `.msi.sig` next to it for
in-app updater verification.

## In-app auto-update

Once installed, ClauDepot checks for updates on its own schedule
(toggle in **Settings → General**). New versions arrive over the
in-app updater channel — signed with the same key the release
workflow embeds at build time, so a download tampered in transit
won't install. The current channel and any pending update show in
**Settings → About**.

## Build from source

Useful if you want to track `main`, run an unreleased feature, or
package for a platform that the release matrix doesn't cover.

Prerequisites:

- A recent **Rust toolchain** ([rustup.rs](https://rustup.rs)).
- **Node 20+** with **pnpm** ([pnpm.io](https://pnpm.io)).

No Xcode-only tooling on macOS, no MSVC on Windows beyond what
`rustup` pulls in, no extra system libraries on Linux.

```bash
git clone https://github.com/xiaolai/claudepot-app
cd claudepot-app
pnpm install

# Command-line tool
cargo build -p claudepot-cli --release
# Binary lands at ./target/release/claudepot

# Desktop app
pnpm tauri build --no-bundle   # raw .app / binary / .exe, no installer
# OR
pnpm tauri dev                 # run the GUI in dev mode (hot reload)
```

`--no-bundle` skips the platform installer and writes the raw bundle
to `src-tauri/target/release/bundle/`. On macOS that's a `.app`; on
Linux it's a binary; on Windows it's an `.exe`. To produce the same
signed installers the release workflow ships, drop the flag — but
note that bundling on macOS / Windows expects the same signing
configuration the release workflow uses, so unless you're set up for
that, the unsigned `--no-bundle` output is what you want locally.

## First run

When you launch ClauDepot the first time, it will:

1. **Check for an existing Claude Code session.** If you've run
   `claude /login` on this machine, ClauDepot finds your current
   credentials and offers to register them as your first account.
2. **Open the Accounts tab.** This is your home base — add other
   accounts here, switch the CLI and Desktop slots independently.
3. **Set up the OS keychain entry.** ClauDepot uses a separate
   keychain item from Claude's own — see the [architecture
   note](https://github.com/xiaolai/claudepot-app/blob/main/.claude/rules/architecture.md)
   for why this split matters.

Your data lives at `~/.claudepot/`. Override with the
`CLAUDEPOT_DATA_DIR` environment variable if your home directory is
on a small SSD.
